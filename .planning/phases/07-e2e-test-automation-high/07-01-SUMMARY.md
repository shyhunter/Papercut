# Phase 07 Plan 01: E2E Infrastructure Scaffold Summary

**One-liner:** WebDriverIO 9 + @crabnebula/tauri-driver E2E scaffold with Linux Xvfb DISPLAY hook, error-path fixture generator, and shared test helpers.

## What Was Built

This plan establishes the complete E2E testing foundation that plans 02 and 03 depend on:

1. **WebDriverIO runner**: `npm run test:e2e` executes the test suite via `wdio run src/e2e/wdio.conf.ts`. The `pretest:e2e` pre-hook auto-generates fixtures before every run.

2. **WDIO config** (`src/e2e/wdio.conf.ts`): Launches the Tauri binary for the current platform (macOS .app bundle, Windows .exe, Linux ELF). Manages `tauri-driver` via `beforeSession`/`afterSession` lifecycle hooks. Sets `DISPLAY=:99` on Linux when no `DISPLAY` env is set (Xvfb support).

3. **Error-path fixtures** (`test-fixtures-e2e/`):
   - `corrupt.pdf` and `corrupt.jpg` â€” committed zero-byte stubs that trigger empty-file errors
   - `large-sparse.pdf` and `large-sparse.jpg` â€” 110 MB sparse files (gitignored, regenerated by `pretest:e2e`) that trigger the >100 MB size guard

4. **Driver helpers** (`src/e2e/helpers/driver.ts`): `prepareOutputPath`, `waitForStep`, `waitForFileLoaded`, `waitForProcessingComplete` (parameterized: `'compare-step'` for PDF, `'image-compare-step'` for images), `screenshotOnFailure`. Exports `E2E_OUTPUT_DIR`, `FIXTURES_DIR`, `REAL_FIXTURES_DIR`.

5. **Dialog mock** (`src/e2e/helpers/dialogs.ts`): `mockSaveDialog(browser, path)` patches `window.__TAURI__.dialog.save` to resolve immediately with a fixed path, bypassing the OS save dialog during tests.

6. **testid for afterEach cleanup**: `data-testid="process-another-btn"` added to the "Process another" button in both `CompareStep.tsx` and `ImageCompareStep.tsx`.

**2. [Rule 1 - Bug] WDIO `services: ['tauri']` does not exist**
- **Found during:** Task 1 (reading @crabnebula/tauri-driver README)
- **Issue:** The plan's `wdio.conf.ts` template used `services: [['tauri', { tauriDriverPath: 'tauri-driver' }]]`. This service does not exist in WDIO v9 or @crabnebula/tauri-driver. The correct pattern is `beforeSession`/`afterSession` lifecycle hooks with `waitTauriDriverReady()`.
- **Fix:** Replaced the `services` entry with `beforeSession`/`afterSession` hooks that spawn `npx tauri-driver` as a child process and poll for readiness.
- **Files modified:** `src/e2e/wdio.conf.ts`
- **Commit:** 246b991

**3. [Rule 1 - Bug] `autoCompileOpts` and `screenshotPath` not in WDIO v9 Testrunner type**
- **Found during:** Task 2 (TypeScript check)
- **Issue:** WDIO v9 removed `autoCompileOpts` from `Options.Testrunner`. TS compilation of `.ts` config files is automatic. `screenshotPath` also does not exist in the type.
- **Fix:** Removed `autoCompileOpts` and `screenshotPath` from the config. Used `WebdriverIO.Config` (which extends `Options.Testrunner` + `WithRequestedTestrunnerCapabilities`) for the config type instead of `Options.Testrunner` alone. Added a comment noting that screenshots are managed by `screenshotOnFailure()` in `driver.ts`.
- **Files modified:** `src/e2e/wdio.conf.ts`
- **Commit:** 246b991
