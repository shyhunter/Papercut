**src/lib/pdfMerge.ts** — Merge engine using pdf-lib:

```ts
interface MergeInput {
  filePath: string;
  fileName: string;
  pageCount: number;
  bytes: Uint8Array;
}

interface MergeResult {
  bytes: Uint8Array;
  totalPages: number;
  sourceFiles: string[];
}
```

`mergePdfs(inputs: MergeInput[]): Promise<MergeResult>`:
1. Create new `PDFDocument.create()`
2. For each input in order: `PDFDocument.load(input.bytes)` → `copyPages` → `addPage` for each copied page
3. Save with `useObjectStreams: true` (same pattern as pdfProcessor — never useCompression per P02 decision)
4. Return bytes + total page count + source file names

Also export a helper `loadPdfForMerge(filePath: string): Promise<MergeInput>`:
1. Read file via Tauri `readFile()`
2. Load with pdf-lib to get page count
3. Extract file name from path
4. Return MergeInput with bytes, fileName, pageCount

**src/hooks/useMergePdfProcessor.ts** — Hook following processor pattern:

```ts
interface UseMergePdfProcessor {
  isProcessing: boolean;
  result: MergeResult | null;
  error: string | null;
  merge: (inputs: MergeInput[]) => Promise<void>;
  reset: () => void;
}
```

Same pattern as `usePdfProcessor`: state for isProcessing/result/error, merge() calls `mergePdfs()` wrapped in try/catch, reset() clears state.
  </action>

`npx tsc --noEmit` passes. Both files compile with correct types.
