**src/types/tools.ts** — Create the tool type system:

```ts
export type ToolId =
  | 'compress-pdf'
  | 'compress-image'
  | 'merge-pdf'
  | 'split-pdf'
  | 'rotate-pdf';

export type ToolCategory = 'pdf' | 'image';

export interface ToolStep {
  label: string;
  description: string;
}

export interface ToolDefinition {
  id: ToolId;
  name: string;
  description: string;
  category: ToolCategory;
  icon: string;           // Lucide icon name (e.g., 'FileDown', 'Merge')
  acceptsFormats: Array<'pdf' | 'image'>;
  steps: ToolStep[];      // The step sequence for this tool's flow
}
```

Create `TOOL_REGISTRY: Record<ToolId, ToolDefinition>` with entries for all 5 tools:
- `compress-pdf`: steps = Pick → Configure → Compare → Save (existing)
- `compress-image`: steps = Pick → Configure → Compare → Save (existing)
- `merge-pdf`: steps = Pick Files → Order → Save
- `split-pdf`: steps = Pick → Select Pages → Save
- `rotate-pdf`: steps = Pick → Select & Rotate → Save

Each tool has `acceptsFormats` for file-drop filtering later:
- compress-pdf, merge-pdf, split-pdf, rotate-pdf → `['pdf']`
- compress-image → `['image']`

**src/context/ToolContext.tsx** — React context for tool routing:

```ts
interface ToolContextValue {
  activeTool: ToolId | null;       // null = on dashboard
  activeToolDef: ToolDefinition | null;
  selectTool: (toolId: ToolId) => void;
  goToDashboard: () => void;
}
```

Implement as a simple React Context + Provider using `useState<ToolId | null>(null)`. `selectTool` sets the tool, `goToDashboard` sets null. `activeToolDef` is derived from TOOL_REGISTRY[activeTool].

Export `ToolProvider` (wraps children) and `useToolContext()` (calls useContext, throws if used outside provider).
