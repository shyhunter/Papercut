**src/lib/pdfRotate.ts** — Rotation engine using pdf-lib:

Types:
```ts
type RotationDegrees = 0 | 90 | 180 | 270;

interface PageRotation {
  pageIndex: number;      // 0-based
  rotation: RotationDegrees;
}

interface RotateResult {
  bytes: Uint8Array;
  totalPages: number;
  rotations: PageRotation[];  // What was applied
}
```

`rotatePdf(pdfBytes: Uint8Array, rotations: PageRotation[]): Promise<RotateResult>`:
1. Load with `PDFDocument.load(pdfBytes)`
2. For each rotation entry: `page = pdfDoc.getPage(index)` → `page.setRotation(degrees(rotation))`
3. pdf-lib `degrees()` helper converts number to rotation object
4. Save with `useObjectStreams: true`
5. Return bytes + totalPages + applied rotations

Helper: `cycleRotation(current: RotationDegrees): RotationDegrees`:
- 0 → 90 → 180 → 270 → 0

**src/hooks/useRotatePdfProcessor.ts** — Hook:

```ts
interface UseRotatePdfProcessor {
  isProcessing: boolean;
  result: RotateResult | null;
  error: string | null;
  rotate: (pdfBytes: Uint8Array, rotations: PageRotation[]) => Promise<void>;
  reset: () => void;
}
```

Same pattern as other processor hooks.

**src/components/rotate/RotateStep.tsx** — Combined select-and-rotate step:

1. **Page grid:** Responsive grid of page thumbnails (same rendering as split — `renderAllPdfPages()` at ~0.4 scale)
   - Each thumbnail shows:
     - The page image with CSS `transform: rotate(Xdeg)` to visually preview the rotation
     - Small rotation badge in the corner showing current degrees (90°, 180°, 270°) — hidden when 0°
   - Click a thumbnail → cycle its rotation (0→90→180→270→0)
   - Selected (non-zero rotation) pages get a highlight border (blue/primary)

2. **Bulk controls toolbar:** Above the grid:
   - "Rotate All →" button (rotate all 90° clockwise)
   - "Rotate All ←" button (rotate all 90° counter-clockwise)
   - "Reset All" button (set all back to 0°)
   - Show count: "X pages rotated"

3. **State:** `rotations: Map<number, RotationDegrees>` — keyed by page index
   - Initial state: all pages at 0° (or read existing rotation from PDF metadata)
   - Clicking thumbnail: `cycleRotation(current)` for that page
   - "Rotate All →": apply +90° to every page
   - "Rotate All ←": apply -90° (which is +270°) to every page
   - "Reset All": set all to 0°

4. **Actions:**
   - "Back" returns to Pick step
   - "Apply & Save" button: converts rotations map to PageRotation[], calls processor hook, advances to Save
   - Disable "Apply & Save" if no pages have been rotated (all at 0°)

5. **Visual design:**
   - Grid: `grid grid-cols-3 sm:grid-cols-4 md:grid-cols-5 lg:grid-cols-6 gap-3`
   - Thumbnail container: `aspect-[3/4] rounded-lg border overflow-hidden cursor-pointer`
   - CSS rotation on the `<img>` inside the container (with `object-fit: contain`)
   - Rotation badge: small absolute-positioned pill in bottom-right corner
