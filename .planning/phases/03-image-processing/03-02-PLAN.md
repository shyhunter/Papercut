  <action>
Create `src/components/ImageConfigureStep.tsx`. Mirror `ConfigureStep.tsx` structure but for image processing.

**Props interface:**
```typescript
export interface ImageConfigureStepProps {
  fileName: string;
  fileSizeBytes: number;      // original source file size (shown in header)
  sourceFormat: ImageOutputFormat;  // detected source format — sets default outputFormat
  isProcessing: boolean;
  error: string | null;
  lastResult: ImageProcessingResult | null;  // used to show estimated output size in slider label
  onGeneratePreview: (options: ImageProcessingOptions) => void;
  onBack: () => void;
}
```

**Quality slider (LOCKED decisions):**
- `<input type="range" min="1" max="100" step="1">` rendered as a slider.
- Fires processing on `onMouseUp` / `onTouchEnd` only — do NOT call onGeneratePreview on `onChange`. The `onChange` handler only updates local slider state (so the thumb moves visually during drag).
- Label logic:
  - When `outputFormat !== 'png'`: show `"{quality}% — ~{estimatedSize}"` where `estimatedSize` is `formatBytes(lastResult.outputSizeBytes)` if `lastResult` exists, else just `"{quality}%"`.
  - When `outputFormat === 'png'`: slider value maps to user-facing "Compression" not "Quality". Display `"Compression: {compressionDisplay}/9"` where `compressionDisplay = Math.round((100 - quality) * 9 / 100)`. File size estimate: same pattern — `"Compression: {N}/9 — ~{estimatedSize}"` if lastResult exists.
  - The `~` prefix signals it's from the previous run, not a live prediction.
- Controls adapt instantly on format change — just re-read from local state.

**Format selector (LOCKED):**
Always visible. Three buttons (or segmented control): JPG | PNG | WebP. Default = `sourceFormat` passed in props.

**Resize section (LOCKED):**
- Same pill toggle as PDF's "Resize pages" — off by default.
- When enabled, show:
  - **Unit toggle**: pixels | percentage — a small two-button toggle, defaults to "pixels".
  - **Preset buttons**: HD 1920×1080 | Web 1280×720 | Square 1080×1080 | Thumbnail 400×400. Clicking a preset fills W and H fields (in pixels; if unit=percentage, convert based on sourceWidth/sourceHeight from lastResult if available — if no result yet, just set unit to "pixels" when preset is selected).
  - **Width and Height inputs**: numeric text inputs (`type="number"`, min="1"). Show placeholder "Width" and "Height" with "px" or "%" unit suffix labels.
  - **Aspect ratio lock toggle** (defaults to unlocked): A small lock icon button. When locked, changing one dimension recomputes the other based on the aspect ratio derived from `lastResult.sourceWidth / lastResult.sourceHeight` (or `1` if no result yet). Use `Lock` and `Unlock` icons from `lucide-react`.
- Invalid dimension inputs (NaN, 0, negative): show a red validation message below the inputs ("Width and height must be positive numbers").

**Submit:**
The "Generate Preview" button triggers `handleSubmit()`:
1. Build `ImageProcessingOptions` from local state.
2. If resize is enabled, validate width and height are positive integers — show inline error, return early if invalid.
3. Call `onGeneratePreview(options)`.

**Disabled state:** All controls disabled when `isProcessing === true`. Show inline spinner or "Processing…" text on the button.

**Error display:** Same pattern as ConfigureStep — inline red bordered paragraph if `error` is non-null.

**Component layout:** Same card layout as ConfigureStep — `flex flex-1 flex-col items-center overflow-y-auto p-6` with `w-full max-w-lg space-y-4 my-auto`. File name + size in header. Two cards: "Image quality" (slider + format) and "Resize" (toggle + expanded controls).

   ```typescript
   function detectImageFormat(filePath: string): ImageOutputFormat {
     const ext = filePath.split('.').pop()?.toLowerCase() ?? '';
     if (ext === 'png') return 'png';
     if (ext === 'webp') return 'webp';
     return 'jpeg'; // jpg and jpeg both map to 'jpeg'
   }
   ```
