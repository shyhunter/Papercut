Output: ToolSidebar component with icon strip, expandable panels, before/after preview, and inline tool execution.

<task type="auto">
  <name>Task 1: ToolSidebar icon strip and expandable panel</name>
  <files>src/components/pdf-editor/ToolSidebar.tsx, src/components/pdf-editor/ToolSidebarPanel.tsx, src/components/pdf-editor/ToolSidebarPreview.tsx, src/types/tools.ts</files>
  <action>
1. Identify PDF-only tools from TOOL_REGISTRY to show in sidebar. Filter: `category === 'pdf'` AND exclude tools that don't make sense inline (e.g., merge-pdf requires multiple files, split-pdf produces multiple outputs). Include:
   - compress-pdf
   - rotate-pdf
   - page-numbers
   - watermark
   - crop-pdf
   - organize-pdf (already have page panel, but include for completeness)
   - sign-pdf
   - redact-pdf
   - pdfa-convert
   - repair-pdf
   - protect-pdf
   - unlock-pdf

   Add a helper function or constant: `EDITOR_SIDEBAR_TOOLS: ToolId[]` listing the above in a logical order.

2. Create `src/components/pdf-editor/ToolSidebar.tsx`:
   - Positioned on the right side of the editor
   - Collapsed state (default): thin icon strip (~48px wide, like VS Code activity bar)
     - Each tool shown as its Lucide icon in a square button, vertically stacked
     - Tooltip on hover showing tool name
     - Active tool icon has a left border highlight (like VS Code)
   - Expanded state: icon strip + adjacent panel (~280px total)
     - Click an icon to expand with that tool's panel
     - Click the same icon again to collapse
     - Click a different icon to switch panels
   - Collapse/expand toggle at the bottom of the icon strip

3. Create `src/components/pdf-editor/ToolSidebarPreview.tsx`:
   - Receives two props: `originalBytes: Uint8Array` (current PDF) and `previewBytes: Uint8Array | null` (result of running tool with current settings)
   - Renders two side-by-side thumbnail images labeled "Before" and "After"
   - Uses `renderPdfPageThumbnail()` from `src/lib/pdfThumbnail.ts` to render the current page (from EditorContext's `currentPage`) as a thumbnail for each
   - If `previewBytes` is null (settings not yet changed or still processing), show the "After" side with a placeholder or dimmed copy of "Before"
   - Thumbnails should be small enough to fit within the ~230px panel width (e.g., ~100px each side-by-side)
   - IMPORTANT: Always pass `bytes.slice()` to pdf.js to avoid detached ArrayBuffer issues (see project memory)

4. Create `src/components/pdf-editor/ToolSidebarPanel.tsx`:
   - Renders inline tool settings based on the selected ToolId
   - For each tool, show a simplified version of its configuration UI:
     - Title + description at top
     - Tool-specific settings (reuse existing ConfigureStep logic where possible)
     - **Before/after preview** (ToolSidebarPreview) between settings and Apply button
     - "Apply" button at bottom
   - **Preview pipeline:** When settings change (debounced, ~500ms), run the tool's processing pipeline on the current pdfBytes to produce `previewBytes`. Pass both `originalBytes` (from EditorContext) and `previewBytes` to ToolSidebarPreview. This lets the user see what the tool will do BEFORE clicking Apply.
   - On "Apply": use the already-computed `previewBytes` (or re-run if stale) → call `updatePdfBytes(newBytes)` on EditorContext → canvas refreshes
   - Show a loading spinner on the "After" thumbnail while the preview pipeline runs
   - Show success/error feedback after Apply

   For this initial plan, implement the inline panel for 2-3 tools as proof of concept:
   - **compress-pdf**: quality preset selector + before/after preview + Apply (invokes compress_pdf Tauri command)
   - **rotate-pdf**: rotation angle selector + page range + before/after preview + Apply
   - **watermark**: text input + opacity/rotation/color + before/after preview + Apply

   The remaining tools can be wired in a follow-up or during integration. Each tool panel follows the same pattern: settings form → debounced preview → before/after thumbnails → Apply → updatePdfBytes.

  <done>Tool sidebar shows PDF tool icons. Clicking expands a settings panel with before/after preview. At least compress, rotate, and watermark show live preview thumbnails before Apply.</done>
