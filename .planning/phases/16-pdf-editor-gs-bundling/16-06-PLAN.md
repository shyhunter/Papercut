  <name>Task 1: File association and launch-into-editor flow</name>
  <files>src-tauri/tauri.conf.json, src-tauri/src/main.rs, src-tauri/src/lib.rs, src/App.tsx, src/context/ToolContext.tsx</files>
  <action>
1. Add PDF file association to tauri.conf.json:
   - Under `bundle`, add file associations:
   ```json
   "fileAssociations": [
     {
       "ext": ["pdf"],
       "mimeType": "application/pdf",
       "description": "PDF Document",
       "role": "Editor"
     }
   ]
   ```
   - On macOS this registers via Info.plist; on Windows via NSIS installer; on Linux via .desktop file.

2. Handle file-open events in Rust backend:
   - In `main.rs` or `lib.rs` setup, check `std::env::args()` for a PDF file path argument (macOS sends the file path as a CLI argument on "Open with").
   - Also listen for the Tauri `tauri://file-drop` or macOS-specific `NSApplication openFile` event for files opened while the app is already running.
   - Emit a custom event `file-opened` to the frontend with the file path: `app.emit("file-opened", payload)`.

3. Handle the `file-opened` event in the frontend:
   - In `App.tsx` or `ToolContext.tsx`, listen for the `file-opened` Tauri event on mount.
   - When received: set a new state `editorFilePath: string | null` in ToolContext.
   - Add a new navigation target: `openEditor(filePath: string)` to ToolContext alongside `selectTool` and `goToDashboard`.
   - When `editorFilePath` is set, render EditorView instead of Dashboard or ToolFlow.

4. Update `App.tsx` routing:
   - If `editorFilePath` is set → render EditorView with that file
   - If `activeTool` is set → render ToolFlow (existing)
   - If neither → render Dashboard (existing)
   - Dashboard should also have an "Open in Editor" option for PDF files (or clicking on the existing "Edit PDF" tool card routes to the new editor)

5. No first-launch prompt for default handler — user registers manually via system Preferences or right-click → "Open with".
  </action>
  <verify>
`cargo check` compiles. Manual test on macOS: right-click a PDF → Open With → Papercut → app opens directly into editor view.
  </verify>
  <done>PDFs opened via system file association launch directly into the editor view. Dashboard still works as entry point for tool selection.</done>

<task type="auto">
  <name>Task 2: Save controller, unsaved changes guard, and editor save flow</name>
  <files>src/components/pdf-editor/SaveController.tsx, src/components/pdf-editor/EditorView.tsx, src/components/pdf-editor/EditorTopToolbar.tsx, src/context/EditorContext.tsx</files>
  <action>
1. Create `src/components/pdf-editor/SaveController.tsx`:
   - Invisible component that registers keyboard shortcuts:
     - Cmd+S (Ctrl+S on Windows/Linux): Save
     - Cmd+Shift+S (Ctrl+Shift+S): Save As
   - Save logic:
     - If `filePath` is null (first save): open Save As dialog (Tauri dialog.save with PDF filter), save bytes, store filePath in EditorContext
     - If `filePath` is set: overwrite the file directly (no dialog)
     - Save As: always opens dialog regardless of existing filePath
   - Before saving: call `applyEdits(pdfBytes, pages)` from `pdfEditor.ts` to generate final PDF bytes with all text/image edits baked in
   - After save: clear `isDirty`, update `filePath` if changed, show success toast
   - During save: brief saving indicator

2. Add unsaved changes guard:
   - In EditorView: intercept navigation attempts (goToDashboard, selectTool, browser close)
   - If `isDirty`: show a confirmation dialog: "You have unsaved changes. Save before leaving?" with Save / Don't Save / Cancel buttons
   - "Save" → run save flow, then navigate
   - "Don't Save" → navigate without saving
   - "Cancel" → stay in editor
   - For app close (window close event): use Tauri's `onCloseRequested` event to intercept and show the same dialog
   - Use `beforeunload` event as well for the webview navigation case

3. Add save button to EditorTopToolbar:
   - Small save icon (floppy disk) next to the breadcrumb
   - Enabled only when `isDirty` is true
   - Shows "Saved" text briefly after successful save

4. Update breadcrumb to show filename:
   - "Dashboard > filename.pdf" (or "Untitled.pdf" if no file path yet)
   - Click "Dashboard" triggers the unsaved changes guard, then navigates
  </action>
