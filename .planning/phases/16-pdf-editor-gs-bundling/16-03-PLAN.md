1. Update EditorContext with text editing state:
   - Add `selectedBlockId: string | null`, `editingBlockId: string | null`, `editorMode: EditorMode` to state
   - Add actions: `selectBlock(id)`, `startEditing(id)`, `stopEditing()`, `updateTextBlock(pageIdx, block)`, `addTextBlock(pageIdx, block)`, `deleteTextBlock(pageIdx, blockId)`, `updatePageEditState(pageIdx, state)`
   - Track text blocks per page in `pages[].textBlocks`

2. Create `src/components/pdf-editor/TextEditingLayer.tsx`:
   - Absolutely positioned overlay on each page in EditorCanvas
   - On mount for each page: call `extractPageText(pdfBytes, pageIndex)` to get text blocks (lazy, cached)
   - Render each text block as a positioned div matching its bounding box (scale with zoom)
   - Click on a text block: select it (blue dashed border, resize handles at corners)
   - Double-click: enter inline editing mode — show a contentEditable div with the text, cursor positioned
   - When editing: typing modifies the text, Enter creates newline (not submit), Escape exits editing
   - Click outside any block: deselect
   - Right-click context menu: Delete, Duplicate (Claude's discretion for exact menu items)
   - Moving selected blocks: drag to reposition (update x/y in block state)
   - Mark the editor as dirty when any text is modified

3. Add "Add Text" button somewhere accessible (e.g., top toolbar or floating action) that inserts a new empty text block at center of current page.

Important: Reuse coordinate/rendering logic from Phase 13's TextOverlay.tsx where applicable. The key difference is this layer works across all pages in continuous scroll, not just one page at a time.
