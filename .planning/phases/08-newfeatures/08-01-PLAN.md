<task type="auto">
  <name>Task 1: Compressibility detection, inline warning, and threshold info</name>
  <files>src/components/ConfigureStep.tsx, src/App.tsx</files>
  <action>
1. Add `compressibilityScore: number` and `imageCount: number` to `ConfigureStepProps`. These come from the pre-scan in pdfProcessor and will be threaded through from App.tsx after getPdfMeta is extended.

2. **Add a new export `getPdfCompressibility` in `src/lib/pdfProcessor.ts`:** This function wraps the private `scanPdfImages` and returns `{ imageCount, compressibilityScore }` given a file path. Pattern: load bytes via `readFile`, open with `PDFDocument.load`, call `scanPdfImages`, return the result. This is analogous to the existing `getPdfImageCount` export but returns the full scan result.

3. In App.tsx, import `getPdfCompressibility` from pdfProcessor.ts and call it in `getPdfMeta`. Store results in state: `const [pdfCompressibility, setPdfCompressibility] = useState<{imageCount: number; compressibilityScore: number}>({imageCount: 0, compressibilityScore: 0})`. Pass to ConfigureStep.

4. In ConfigureStep, add an always-visible info box inside the compression card (below the quality selector, above target size input):
   - Show compressibility guidance text:
     - compressibilityScore >= 0.5: "This PDF contains images — compression will reduce file size significantly."
     - compressibilityScore 0.1–0.49: "This PDF contains some images — moderate compression savings expected."
     - compressibilityScore < 0.1: "This PDF is mostly text — compression savings will be minimal."
   - Use a subtle info style: `text-xs text-muted-foreground` with a small info icon (from lucide-react: `Info`).

5. When `compressibilityScore < 0.1` (text-only PDF), show an inline warning box:
   - Yellow/amber border: `border-amber-500/40 bg-amber-500/10`
   - Text: "This file is mostly text with no embedded images. Compression presets have minimal effect on text-only PDFs."
   - **Block the Generate Preview button** by disabling it and changing label to "Compression not available" when compressibilityScore < 0.1 AND resizeEnabled is false. If resizeEnabled is true, allow progression (resize still useful even for text-only PDFs).

6. The block logic: `const isNonCompressible = compressibilityScore < 0.1 && !resizeEnabled;` — use this to disable the Generate Preview button.
  </action>

- Visual: text-only PDF shows amber warning + disabled Generate Preview; image-heavy PDF shows positive guidance
  </verify>
  <done>
- Non-compressible PDFs show inline warning and cannot proceed to Compare
- Compressibility threshold info is always visible with contextual guidance
- Resize-only flow still works for text-only PDFs

<task type="auto">
  <name>Task 2: Custom quality option with MB/KB target size input and validation</name>
  <files>src/components/ConfigureStep.tsx, src/types/file.ts</files>
  <action>
1. In `src/types/file.ts`, extend `PdfQualityLevel` to include `'custom'`: `export type PdfQualityLevel = 'web' | 'screen' | 'print' | 'archive' | 'custom';`

2. In ConfigureStep, add a 5th radio option to QUALITY_LEVELS array:
   ```
   { value: 'custom', label: 'Custom', description: 'Set your own target file size' }
   ```
   Change the grid from `grid-cols-4` to `grid-cols-5`.

3. When `qualityLevel === 'custom'`, show a target size input section (replaces/augments the existing target size input):
   - Integer input field (`type="number"`, min=1, step=1) for the target value
   - MB/KB toggle button that auto-selects based on `fileSizeBytes`: `fileSizeBytes >= 1024*1024 ? 'MB' : 'KB'`
   - Store as state: `const [customUnit, setCustomUnit] = useState<'MB'|'KB'>(fileSizeBytes >= 1024*1024 ? 'MB' : 'KB')`
   - Store custom value: `const [customSizeValue, setCustomSizeValue] = useState<string>('')`

4. Validation on submit (in handleSubmit):
   - Parse custom size: `const customBytes = parseInt(customSizeValue) * (customUnit === 'MB' ? 1024*1024 : 1024)`
   - If `customBytes >= fileSizeBytes`: show error "Target must be smaller than original ({formatBytes(fileSizeBytes)})"
   - If empty or invalid: show error "Enter a valid target size"
   - On valid: use `recommendQualityForTarget(customBytes, fileSizeBytes, compressibilityScore)` to derive the actual quality level, then pass that derived level (not 'custom') in the options to onGeneratePreview. Set `targetSizeBytes` to `customBytes`.

5. When custom is selected, hide the existing free-text target size input (the "Target size (optional)" field) since custom IS the target size mechanism. When any of the 4 preset qualities is selected, show the existing target size input as before (optional).

6. In `src/lib/pdfProcessor.ts`, add `custom: 'screen'` placeholder entry to `QUALITY_TO_GS_PRESET` to satisfy the `Record<PdfQualityLevel, string>` constraint after adding `'custom'` to the type. This placeholder should never actually be reached — add a runtime guard in `processPdf`: if `options.qualityLevel === 'custom'` throw an error "Custom quality must be resolved to a preset before processing". The guard runs BEFORE the preset lookup.
  </action>

- Visual: 5th Custom radio button appears; selecting it shows integer input + MB/KB toggle; validation blocks invalid sizes
  </verify>

  </done>
