The changes are in different sections of App.tsx and should not conflict. Additionally, when "Start Over" is clicked (whether named by 08-02 or still "Process another"), clear `savedFilePath` state.

1. **Add `shell:allow-open` capability** to `src-tauri/capabilities/default.json` permissions array. This allows `open()` from `@tauri-apps/plugin-shell` to open files in the default OS app (e.g., Preview for PDFs).

2. **Create a SaveConfirmation component** (inline in SaveStep.tsx or separate — inline is fine for scope):
   - Renders as a card at the top of the view: `rounded-lg border border-border bg-card shadow-sm p-4 mx-4 mt-3`
   - Layout: animated checkmark icon | text block | X close button
   - Text block: "File saved successfully" (bold) + saved file path (muted, truncated with `truncate` class)
   - **Clickable file link**: The file path is a clickable link that calls `open()` from `@tauri-apps/plugin-shell` to open the file in the default OS app. Style as a blue link: `text-primary underline cursor-pointer hover:text-primary/80`. Wrap in a button/anchor-like element.
   - **X close button**: Top-right corner, `absolute top-2 right-2`, uses lucide-react `X` icon. On click, calls `onDismiss` prop.
   - The card is `relative` for the absolute X positioning.

3. **Update App.tsx save flow:**
   - Instead of showing a toast on save success, store the saved path in state: `const [savedFilePath, setSavedFilePath] = useState<string | null>(null)`
   - In the `onSaveComplete` callback: `setSavedFilePath(savedPath); setCurrentStep(2);` (go back to Compare but with confirmation visible)
   - Render the SaveConfirmation card above the Compare step when `savedFilePath` is not null:
     ```
     {savedFilePath && (
       <SaveConfirmation
         savedPath={savedFilePath}
         onDismiss={() => setSavedFilePath(null)}
       />
     )}
     ```
   - Place this rendering just after StepBar and before the step content, so it appears at the top.
   - Clear `savedFilePath` when user clicks "Start Over" or "Back" from Compare.
   - Remove the `toast.success('File saved', ...)` calls since the card replaces the toast.

4. **The file open action:**
   ```typescript
   import { open } from '@tauri-apps/plugin-shell';
   // In the click handler:
   await open(savedPath);
   ```
   This opens the file with the OS default handler (Preview for PDF, default image viewer for images).

1. **Create a CSS-based animated checkmark** inside the SaveConfirmation card (left side, before text).
   - Use a combination of CSS keyframe animations — no external library needed.
   - Structure: a circular container with a checkmark SVG inside.

2. **Animation sequence (CSS keyframes):**
   - `@keyframes checkmark-circle`: Circle draws in (stroke-dashoffset animation from full to 0), 0.4s ease-in-out
   - `@keyframes checkmark-check`: Checkmark stroke draws in after circle, 0.3s ease-in-out with 0.3s delay
   - `@keyframes checkmark-glow`: Subtle green glow pulse that plays once: `box-shadow: 0 0 0 0 rgba(34, 197, 94, 0.5)` -> `0 0 20px 10px rgba(34, 197, 94, 0)`, 0.6s with 0.5s delay
   - `@keyframes sparkle-fade`: Small sparkle dots (pseudo-elements) scale up and fade out, 0.5s with 0.6s delay

3. **Implementation:**
   - Add the keyframe definitions as a `<style>` tag in the component (or in App.css / a new CSS module).
   - The animated container: `w-10 h-10 flex-none` with the circle/check SVG.
   - SVG approach: `<svg viewBox="0 0 52 52">` with a `<circle>` (green stroke, animated dash) and a `<path>` (checkmark, animated dash).
   - Apply `animation` properties to the SVG elements.
   - The glow effect is on the outer container div.

4. **Keep it tasteful:** The animation plays once on mount (not looping). Total animation duration: ~1.2s. After animation completes, the checkmark remains static (green circle with white checkmark).

5. **Color:** Green tones — `stroke: #22c55e` (green-500) for the circle and check. Glow uses the same green with opacity.
