<task type="auto">
  <name>Task 1: Responsive root font-size and CSS scaling foundation</name>
  <files>src/App.css, src/styles/globals.css</files>
  <action>
1. **Replace the fixed root font-size** in `src/App.css`:
   - Current: `font-size: 18px` (fixed)
   - New: `font-size: clamp(14px, 1.2vw + 0.4vh, 20px);`
   - This gives: ~14px at 900px window width (min floor), ~16-17px at 1100px (default), ~20px at large windows
   - All rem-based sizes throughout the app (Tailwind's spacing, text sizes) automatically scale since Tailwind uses rem.

2. **Add responsive CSS custom properties** in `src/App.css` under `:root`:
   ```css
   :root {
     /* Responsive scaling — drives all proportional sizing */
     --scale-factor: clamp(0.85, 0.065vw + 0.7, 1.15);
     /* Used for explicit scaling of non-rem elements */
     --icon-size: clamp(14px, 1.2vw, 22px);
     --button-padding-x: clamp(0.6em, 0.8vw + 0.3em, 1.4em);
     --button-padding-y: clamp(0.3em, 0.4vw + 0.2em, 0.8em);
     --card-padding: clamp(12px, 1.2vw, 20px);
     --gap-sm: clamp(4px, 0.4vw, 8px);
     --gap-md: clamp(8px, 0.8vw, 16px);
     --gap-lg: clamp(12px, 1.2vw, 24px);
   }
   ```

3. **Update the global `input, button` styles** in `src/App.css`:
   - Change `padding: 0.6em 1.2em` to `padding: var(--button-padding-y) var(--button-padding-x)`
   - Change `font-size: 1em` to `font-size: inherit` (inherits from root clamp)

4. **In `src/styles/globals.css`**, add responsive body styling:
   ```css
   body {
     font-size: clamp(14px, 1.2vw + 0.4vh, 20px);
     line-height: 1.5;
   }
   ```
   This ensures the base font-size is set even if App.css :root is overridden.

5. **Key principle:** Since Tailwind v4 uses rem units for spacing (p-4 = 1rem, text-sm = 0.875rem, etc.), changing the root font-size automatically scales all Tailwind utility classes. This is the highest-leverage change — most components scale automatically without per-component changes.
  </action>
  <verify>
- `npx tsc --noEmit` passes (CSS changes only, no TS impact)
- Visual: resize window from 900px to 1400px width — fonts, spacing, buttons all scale smoothly
- At minWidth (900px): UI is compact but fully readable
- At default size (1100px): UI looks identical to current design
- At large size (1400px+): UI scales up comfortably without breaking layout
  </verify>
  <done>
- Root font-size uses CSS clamp() with viewport-relative units
- All Tailwind rem-based sizes scale automatically
- CSS custom properties available for explicit scaling
  </done>
