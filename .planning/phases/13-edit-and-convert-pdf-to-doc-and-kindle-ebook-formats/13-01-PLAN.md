  <action>
1. Create `src/types/editor.ts` with PDF editor types:
   - `TextBlock`: id, pageIndex, x, y, width, height, text, fontSize, fontName, color, alignment ('left'|'center'|'right'), isNew (boolean for newly added blocks)
   - `ImageBlock`: id, pageIndex, x, y, width, height, imageBytes (Uint8Array), rotation (0|90|180|270), flipH (boolean), flipV (boolean), isNew (boolean)
   - `PageEditState`: pageIndex, textBlocks (TextBlock[]), imageBlocks (ImageBlock[]), deletedTextIds (string[]), deletedImageIds (string[])
   - `EditorState`: pdfBytes (Uint8Array), pages (PageEditState[]), currentPage (number), isDirty (boolean)
   - `EditorMode`: 'select' | 'text' | 'image'

2. Create `src/types/converter.ts` with conversion types:
   - `ConvertFormat`: 'docx' | 'doc' | 'odt' | 'epub' | 'mobi' | 'azw3' | 'txt' | 'rtf' | 'pdf'
   - `EpubLayout`: 'reflowable' | 'fixed'
   - `ConvertOptions`: outputFormat (ConvertFormat), fontFamily (string, optional), fontSize (number, optional), marginTop/Right/Bottom/Left (number, optional), lineSpacing (number, optional), paragraphSpacing (number, optional), epubLayout (EpubLayout, optional)
   - `ConvertResult`: outputBytes (Uint8Array), outputFormat (ConvertFormat), originalSize (number), outputSize (number)
   - `ConverterEngine`: 'libreoffice' | 'calibre' | 'js' (for tracking which engine handles which format)
   - `DOCUMENT_FORMATS`: readonly array of ConvertFormat for LibreOffice-handled formats ('docx', 'doc', 'odt', 'txt', 'rtf', 'pdf')
   - `EBOOK_FORMATS`: readonly array for Calibre-handled formats ('epub', 'mobi', 'azw3')

3. Update `src/types/file.ts`:
   - Add 'document' to `SupportedFormat` union type (currently 'pdf' | 'image')
   - The 'document' format covers DOCX, DOC, ODT, EPUB, MOBI, AZW3, TXT, RTF inputs

4. Update `src/lib/fileValidation.ts`:
   - Add to SUPPORTED_EXTENSIONS: 'docx', 'doc', 'odt', 'epub', 'mobi', 'azw3', 'txt', 'rtf'
   - Extend `detectFormat()`: return 'document' for these new extensions
   - Keep existing 'pdf' and 'image' detection unchanged

5. Update `src/types/tools.ts`:
   - Add 'edit-pdf' | 'convert-doc' to ToolId union
   - Add 'document' to ToolCategory: 'pdf' | 'image' | 'document'
   - Add 'document' to acceptsFormats array type
   - Add TOOL_REGISTRY entries:
     - 'edit-pdf': name "Edit PDF", category 'pdf', icon 'FileEdit', acceptsFormats ['pdf'], steps: Pick -> Edit -> Save
     - 'convert-doc': name "Convert Document", category 'document', icon 'ArrowLeftRight', acceptsFormats ['pdf', 'document'], steps: Pick -> Configure -> Compare -> Save
  </action>

<task type="auto">
  <name>Task 2: Rust sidecar commands for LibreOffice and Calibre</name>
  <files>
    src-tauri/src/lib.rs
    src-tauri/capabilities/default.json
  </files>
  <action>
1. Add to `src-tauri/src/lib.rs` two new Tauri commands following the existing `compress_pdf` spawn+event-loop pattern:

   **convert_with_libreoffice:**
   - Parameters: app (AppHandle), source_path (String), output_format (String)
   - Validate output_format against allow-list: ["docx", "doc", "odt", "pdf", "txt", "rtf"]
   - Create temp dir: `std::env::temp_dir().join("papercut_convert")`
   - Use `app.shell().command("soffice")` (NOT sidecar — LibreOffice is system-installed, not bundled)
   - Args: `["--headless", "--convert-to", &output_format, "--outdir", &tmp_dir_str, &source_path]`
   - Use .spawn() + event loop (same pattern as compress_pdf) to enable future cancellation
   - On success: construct output filename (same stem + new extension), read output bytes with std::fs::read, clean up temp files, return Response::new(bytes)
   - On failure: return Err with descriptive message including "LibreOffice not found" hint if command not found
   - Note: On macOS, soffice is at `/Applications/LibreOffice.app/Contents/MacOS/soffice` — try this path if "soffice" fails

   **convert_with_calibre:**
   - Parameters: app (AppHandle), source_path (String), output_format (String), extra_args (Vec<String>)
   - Validate output_format against allow-list: ["epub", "mobi", "azw3", "pdf"]
   - Create temp output path: temp_dir + source stem + "." + output_format
   - Use `app.shell().command("ebook-convert")` (system-installed)
   - Args: `[source_path, output_path, ...extra_args]`
   - Use .spawn() + event loop pattern
   - On success: read output bytes, clean up, return Response::new(bytes)
   - On failure: return Err with "Calibre ebook-convert not found" hint

2. Register both commands in the `.invoke_handler(tauri::generate_handler![...])` list.

3. Update `src-tauri/capabilities/default.json`:
   - Add shell:allow-execute permission if not already present (needed for shell().command() on system binaries)
   - The existing shell:allow-spawn-sidecar covers sidecar() calls but command() may need shell:allow-execute or the broader shell:default scope

Important: Use `app.shell().command()` not `app.shell().sidecar()` — LibreOffice and Calibre are system-installed, too large to bundle (~200-500MB each). Research confirmed this approach.
  </action>
  <verify>Run `cd src-tauri && cargo check` — Rust compiles without errors. Grep for "convert_with_libreoffice" and "convert_with_calibre" in lib.rs confirms both commands exist and are registered.</verify>
  <done>Both Rust sidecar commands compile, validate inputs, invoke system binaries via shell().command(), and are registered in the invoke handler.</done>
