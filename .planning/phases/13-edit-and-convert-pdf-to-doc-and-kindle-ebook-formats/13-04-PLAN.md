  <action>
Create `src/lib/pdfEditor.ts` with the overlay+redraw save engine:

1. **applyTextEdits(pdfBytes: Uint8Array, pageEdits: PageEditState[]): Promise&lt;Uint8Array&gt;**
   - Load PDF with PDFDocument.load(pdfBytes)
   - For each page with edits:
     - Get the pdf-lib page object
     - **Modified text blocks** (blocks where text or formatting changed from original):
       - Draw a white rectangle over the original text position: page.drawRectangle({ x, y, width, height, color: rgb(1,1,1), borderWidth: 0 })
       - Draw new text at the same position: page.drawText(newText, { x, y, size: fontSize, font, color: rgb(r,g,b) })
       - For alignment: adjust x position (left=original x, center=original x + width/2 - textWidth/2, right=original x + width - textWidth)
     - **Deleted text blocks**: Draw white rectangle over the original position (cover but don't redraw)
     - **New text blocks** (isNew=true): Draw text at the specified position (no white rect needed since there's nothing underneath)
   - Font handling:
     - For standard PDF font names, use PDFDocument.embedFont(StandardFonts.XXX) — map font names: Helvetica, TimesRoman, Courier
     - For unknown fonts, fall back to Helvetica (per research recommendation)
   - Save with { useObjectStreams: false } (consistent with existing pdfProcessor pattern)
   - Return new PDF bytes

2. **mapFontName(pdfFontName: string): StandardFonts**
   - Map common PDF font names to pdf-lib StandardFonts enum:
     - Helvetica* -> StandardFonts.Helvetica (or HelveticaBold, HelveticaOblique)
     - TimesRoman* / Times-Roman* -> StandardFonts.TimesRoman
     - Courier* -> StandardFonts.Courier
     - Default fallback: StandardFonts.Helvetica

3. **hexToRgb(hex: string): {r: number, g: number, b: number}**
   - Convert hex color to 0-1 range RGB values for pdf-lib's rgb() function

Key constraint per research: pdf-lib CANNOT edit existing text in content streams. The white-rectangle + drawText approach is the industry standard for browser-based PDF editors. This is NOT a limitation — it's the correct pattern.
