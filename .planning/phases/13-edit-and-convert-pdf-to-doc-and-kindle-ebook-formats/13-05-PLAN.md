  <name>Task 1: PDF image extraction and image overlay component</name>
  <files>
    src/lib/pdfImageExtract.ts

  <action>
1. **pdfImageExtract.ts** — Extract images from PDF pages:

   **extractPageImages(pdfBytes: Uint8Array, pageIndex: number): Promise&lt;ImageBlock[]&gt;**
   - Load PDF with PDFDocument.load(pdfBytes) from pdf-lib
   - Get page, access page.node.Resources() to find XObject dictionary
   - Iterate XObject entries, filter for Image subtype
   - For each image XObject:
     - Extract raw image bytes from the stream
     - Determine image type from the stream's Filter entry (DCTDecode=JPEG, FlateDecode+DecodeParms=PNG-like)
     - For JPEG images: use raw bytes directly (browser can display)
     - For other formats: render the page region to canvas via pdfjs-dist to capture the image visually (more reliable than manual decode)
     - Get image dimensions: Width, Height from the XObject dict
     - Determine position on page by scanning the content stream for the image draw operator (cm + Do) — this gives the transform matrix including position and scale
     - Build ImageBlock: id (randomUUID), pageIndex, x, y, width, height, imageBytes (as PNG/JPEG Uint8Array), rotation=0, flipH=false, flipV=false, isNew=false
   - If content stream parsing for position is too complex, fall back to: render the full page, diff against page-without-images to identify image regions. Simpler alternative: use pdfjs-dist's OperatorList to find paintImageXObject operators which give position data.
   - Return array of ImageBlocks

   **Note on complexity:** Image position extraction from PDF content streams is non-trivial. If the operator list approach is too fragile, use a simpler heuristic: place images at their XObject dimensions centered on the page, and let users reposition them. The primary value is insert/delete/resize, not pixel-perfect extraction of existing positions.

2. **ImageOverlay.tsx** — Interactive image overlays on canvas:
   - Props: imageBlocks (ImageBlock[]), scale (number), onImageChange (id, updates: Partial<ImageBlock>), onImageDelete (id), onImageInsert (ImageBlock), selectedId (string|null), onSelect (id|null)
   - Render each image as an absolutely-positioned div with an `<img>` element inside:
     - Position: left = block.x * scale, top = (pageHeight - block.y - block.height) * scale (PDF to CSS coords)
     - Size: width = block.width * scale, height = block.height * scale
     - CSS transform for rotation and flip: `rotate(${block.rotation}deg) scaleX(${block.flipH ? -1 : 1}) scaleY(${block.flipV ? -1 : 1})`
   - **Selection**: Click image -> select it (blue border, show handles)
   - **Resize handles**: 8 corner/edge handles (4 corners + 4 midpoints). Drag corner handles to resize with aspect ratio lock (hold Shift to unlock). Use onMouseDown + onMouseMove + onMouseUp pattern.
   - **Drag to move**: When dragging the image body (not a handle), reposition the image. Constrain to page bounds.
   - **Rotation indicator**: Small rotation icon at top-center when selected. Could also be toolbar-only (see EditorToolbar).
   - Image src: create object URL from imageBytes (Blob URL), revoke on unmount or block change
   - Click outside all images -> deselect (onSelect(null))
  </action>
  <verify>Run `npx tsc --noEmit` — no type errors. Open a PDF with images in Edit PDF — images appear as overlays. Click an image — resize handles appear. Drag to move — image repositions. Drag corner handle — image resizes maintaining aspect ratio.</verify>
  <done>Image extraction produces ImageBlocks from PDF pages, image overlays render at correct positions, drag/resize/select interactions work.</done>
</task>

   - **Mode toggle**: Add 'image' mode button alongside existing 'select' and 'text' modes

2. **Update pdfEditor.ts** — Add image editing save:

   **applyImageEdits(pdfBytes: Uint8Array, pageEdits: PageEditState[]): Promise&lt;Uint8Array&gt;**
   - Load PDF with PDFDocument.load(pdfBytes)
   - For each page with image edits:
     - **Deleted images**: Cover the original image area with a white rectangle (same pattern as text deletion — simpler and more reliable than trying to remove the image from the content stream)
     - **Modified images** (moved, resized, rotated): Cover original position with white rect, then embed modified image:
       - Apply rotation/flip to image bytes using canvas (create offscreen canvas, apply transforms, draw image, export as PNG)
       - Embed with pdfDoc.embedPng() or pdfDoc.embedJpg() depending on image type
       - Draw at new position/size: page.drawImage(embeddedImg, { x, y, width, height })
     - **New images** (isNew=true): Embed and draw at specified position/size
   - Important: Image coordinates in ImageBlock are in PDF points (origin bottom-left). pdf-lib drawImage also uses bottom-left origin, so coordinates pass through directly.

   **applyAllEdits(pdfBytes: Uint8Array, pageEdits: PageEditState[]): Promise&lt;Uint8Array&gt;**
   - Combines text and image edits in a single pass:
     - First apply text edits (white rects + drawText)
     - Then apply image edits (white rects + drawImage)
     - Order matters: text first, then images, so images can overlap text if positioned that way
   - This is the main save function called by EditPdfFlow

3. **Update EditorLayout.tsx**:

   - Store image blocks in EditorState.pages[currentPage].imageBlocks
   - Render ImageOverlay as child of PageCanvas (alongside TextOverlay)
   - Wire image toolbar controls to EditorState updates
   - When mode is 'image', show image-specific cursor (crosshair for insert)
   - "Save" button in the editor: calls applyAllEdits() to produce modified PDF bytes, then advances to Save step
  </action>
