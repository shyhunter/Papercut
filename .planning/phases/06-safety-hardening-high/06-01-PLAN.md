  <behavior>
    File size guard (hard cap):
    - FILE_SIZE_LIMIT_BYTES = 100 * 1024 * 1024 (100 MB)
    - getFileSizeBytes(filePath: string): Promise<number> — reads file stat via Tauri readFile, returns byteLength
    - In App.tsx handleFileSelected: after format validation, call getFileSizeBytes. If > FILE_SIZE_LIMIT_BYTES, set fileSizeLimitError state (the size in bytes) — do NOT advance to Configure.
    - LandingCard receives fileSizeLimitBytes?: number | null prop. When non-null, renders a modal dialog (using shadcn Dialog or simple inline overlay) that blocks interaction. Modal content:
      - Header: "File too large"
      - Body: "This file is [X MB]. Files over 100 MB are not supported. Please use a smaller file."
      - Single button: "Cancel" — clears the modal state, returns to idle landing
      - No "proceed anyway" option (locked decision)
    - Modal is a React dialog overlay rendered inside LandingCard or App.tsx — Claude's discretion on exact placement. Use shadcn Dialog component if it is already installed; otherwise build a simple overlay with Tailwind.

    Zero-byte and corrupt file errors:
    - Zero-byte: when getFileSizeBytes returns 0, show inline error (same pattern as invalidDropError): "This file is empty. Please try a different file."
    - Corrupt: the existing corrupt-file handling in App.tsx resets to landing + shows a toast. Change the toast to instead set an inline `fileReadError` state on LandingCard with: "This file appears to be corrupt. Please try a different file." Auto-clear this error in 2.5 s (same pattern as invalidDropError).
    - Mid-flow processing errors (file passes load but fails during processing) already navigate back to landing with a toast. Keep the toast for mid-flow errors; the inline messages are only for load-time failures.

    Cases → expected behavior:
    - 0 bytes → inline: "This file is empty. Please try a different file."
    - > 100 MB → modal: "Files over 100 MB are not supported."
    - supported format, loads OK → advance to Configure (existing)
    - corrupt (fails readFile/parse on load) → inline: "This file appears to be corrupt. Please try a different file."
  </behavior>
