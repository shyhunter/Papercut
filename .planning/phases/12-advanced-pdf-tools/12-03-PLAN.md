<task type="auto">
  <name>Task 1: Signature creation components (Canvas, Typed, Upload)</name>
  <files>src/components/sign-pdf/SignatureCanvas.tsx, src/components/sign-pdf/SignatureTyped.tsx, src/components/sign-pdf/SignatureUpload.tsx</files>
  <action>
    **SignatureCanvas.tsx:**
    - Freehand drawing canvas component (600x200 logical pixels, scaled for device pixel ratio but capped at 2x to avoid memory issues per research pitfall 6)
    - Props: onComplete (dataUrl: string) => void, onClear () => void
    - Uses quadratic Bezier curve interpolation for smooth lines (quadraticCurveTo with point averaging -- see research code example)
    - Store drawn paths in React state (Point[][] array) so canvas can re-render from state on StrictMode remount (research pitfall 4)
    - Black pen, 2px width, round line cap/join
    - "Clear" button to reset all paths
    - Mouse AND touch event handlers (onMouseDown/Move/Up + onTouchStart/Move/End) for both desktop and potential touch input
    - "Use This Signature" button that exports canvas to PNG data URL (cropped to bounding box of drawn content to minimize image size per research pitfall 6)
    - Canvas has a light gray dashed border and a horizontal guideline at ~70% height

    **SignatureTyped.tsx:**
    - Props: onComplete (dataUrl: string) => void
    - Text input for typing name/signature text
    - Font selector with 4 options: "Dancing Script" (flowing), "Caveat" (casual), "Great Vibes" (formal), system monospace as fallback. Use the font-family names from fonts.css.
    - Live preview of text in selected font at signature-appropriate size (rendered in a preview div)
    - "Use This Signature" button that renders the text to an offscreen canvas (at 600x200, using the selected font) and exports as PNG data URL
    - Font size auto-adjusts based on text length to fit within canvas width

    **SignatureUpload.tsx:**
    - Props: onComplete (dataUrl: string) => void
    - File input accepting image formats (PNG, JPG, WebP)
    - Uses Tauri open dialog (plugin-dialog) with image filters
    - Reads the selected image via plugin-fs readFile
    - Shows image preview
    - "Use This Signature" button that converts to PNG data URL (if not already PNG, use canvas to convert)
    - Resize if image is larger than 600x200 to keep embedded signature size reasonable
  </action>
  <verify>Run `npx tsc --noEmit`. Each component can be rendered in isolation (import in a test page).</verify>
  <done>Three signature creation methods produce PNG data URLs. Canvas draws smoothly. Typed text renders in signature fonts. Upload accepts and previews images.</done>
</task>

<task type="auto">
  <name>Task 2: useSavedSignatures hook and SignatureCreateStep</name>
  <files>src/hooks/useSavedSignatures.ts, src/components/sign-pdf/SignatureCreateStep.tsx</files>
  <action>
    **useSavedSignatures.ts:**
    Follow the useRecentDirs hook pattern exactly (LazyStore from @tauri-apps/plugin-store).
    - Store key: 'saved-signatures'
    - Max 10 saved signatures
    - Interface SavedSignature: { id: string, name: string, type: 'drawn' | 'typed' | 'uploaded', dataUrl: string, createdAt: number }
    - Functions: loadSignatures(), saveSignature(sig), deleteSignature(id)
    - IMPORTANT: Always call store.save() after store.set() (per project decision -- no auto-persist)
    - Returns { signatures, saveSignature, deleteSignature, isLoading }

    **SignatureCreateStep.tsx:**
    - Props: onSignatureSelected (dataUrl: string) => void, onBack () => void
    - Two sections:
      1. **Saved Signatures** (top): Grid/list of previously saved signatures with thumbnail preview, name, delete button. Click to select and proceed. Shows "No saved signatures" if empty.
      2. **Create New** (bottom): Tab interface with 3 tabs -- "Draw", "Type", "Upload" -- each rendering SignatureCanvas, SignatureTyped, or SignatureUpload respectively.
    - When a creation component fires onComplete:
      - Prompt for a name (simple inline text input, default to "Signature {N}")
      - Save to local storage via useSavedSignatures
      - Call onSignatureSelected with the data URL to proceed to placement step
    - When a saved signature is clicked: call onSignatureSelected with its dataUrl
    - Visual: Clean card-based layout matching the project's existing Tailwind styling (border-border, bg-card, text-foreground, etc.)
  </action>
  <verify>Run `npx tsc --noEmit`. Create a signature via each method, verify it appears in saved list on re-render.</verify>
  <done>SignatureCreateStep shows saved signatures and allows creating new ones via draw/type/upload. Signatures persist to local storage across sessions.</done>
</task>
