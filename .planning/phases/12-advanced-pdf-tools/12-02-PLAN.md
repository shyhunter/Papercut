<task type="auto">
  <name>Task 1: Add convert_pdfa and repair_pdf Rust commands</name>
  <files>src-tauri/src/lib.rs</files>
  <action>
    Add two new Tauri commands following the protect_pdf pattern exactly.

    **convert_pdfa command:**
    - Parameters: app (AppHandle), source_path (String), pdfa_level (String)
    - Validate pdfa_level is one of "1", "2", "3" (allow-list like compress_pdf presets)
    - Generate a minimal PDFA_def.ps file in temp dir with required pdfmark metadata
    - Spawn GS sidecar with args: -sDEVICE=pdfwrite, -dNOPAUSE, -dBATCH, -dQUIET, -dPDFA={level}, -dPDFACompatibilityPolicy=1, -sColorConversionStrategy=RGB, -sOutputFile={tmp}, {pdfa_def_path}, {source_path}
    - Use the same event loop pattern as protect_pdf (collect stderr, wait for termination)
    - On success: read output file, delete temp files, return bytes
    - On failure: delete temp files, return error with stderr
    - Clean up both PDFA_def.ps and output temp file

    **repair_pdf command:**
    - Parameters: app (AppHandle), source_path (String)
    - Spawn GS sidecar with args: -sDEVICE=pdfwrite, -dNOPAUSE, -dBATCH, -dQUIET, -sOutputFile={tmp}, {source_path}
    - CRITICAL DIFFERENCE from protect_pdf: handle partial success per user decision
    - If GS exits non-zero BUT output file exists with non-zero size: return the bytes as success (not error). The TS side will show a "repaired with potential issues" message.
    - If GS exits non-zero AND no output (or empty): return error with stderr
    - If GS exits 0: return bytes normally
    - Clean up temp file after reading

    Register both commands in the invoke_handler list alongside existing commands.
  </action>
  <verify>Run `cd src-tauri && cargo check` -- no compilation errors. Verify both commands appear in invoke_handler.</verify>
  <done>convert_pdfa and repair_pdf commands compile and are registered. repair_pdf handles partial success (non-zero exit with output).</done>
