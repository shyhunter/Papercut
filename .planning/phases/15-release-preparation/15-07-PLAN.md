<task type="auto">
  <name>Task 2: Opt-in crash reporter</name>
  <files>
    src/components/CrashReporter.tsx
    src/components/ErrorBoundary.tsx
  </files>
  <action>
    1. **CrashReporter component:**
       - Displayed when an unhandled error occurs (inside ErrorBoundary fallback)
       - Shows the user:
         - "Something went wrong" message
         - Error details in a collapsible section (error message, component stack)
         - "Send Crash Report" button — opens GitHub issue URL with error details pre-filled
         - "Dismiss" button — just closes/resets the error boundary
       - The "Send Crash Report" button uses the same GitHub issue URL pattern as FeedbackButton:
         - Label: `bug,crash-report`
         - Body includes: error message, component stack (truncated to 500 chars), app version, OS
       - Privacy: Show the user exactly what will be sent before they click send
       - Store user preference for crash reporting in `papercut-settings.json` via LazyStore (opt-in toggle in About dialog)

    2. **Extend ErrorBoundary:**
       - Update `AppErrorBoundary` and `StepErrorBoundary` in `ErrorBoundary.tsx` to use CrashReporter as their fallback UI instead of the current basic error display
       - Keep the "Try Again" / reload functionality alongside the crash report option
       - StepErrorBoundary: show CrashReporter + "Try Again" (resets to previous step)
       - AppErrorBoundary: show CrashReporter + "Reload App" (window.location.reload)

    Privacy requirements (non-negotiable):
    - Never send anything automatically
    - Show user exactly what will be sent before sending
    - "Send" button opens browser — user must click submit on GitHub too
    - No telemetry, no analytics, no automatic crash collection
  </action>
