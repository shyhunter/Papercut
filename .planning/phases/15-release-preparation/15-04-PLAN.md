<task type="auto">
  <name>Task 1: Create CI validation workflow</name>
  <files>
    .github/workflows/ci.yml
  </files>
  <action>
    Create `.github/workflows/ci.yml` — fast validation on every push and PR:

    1. **Triggers:** `push` (all branches), `pull_request` (all branches)
    2. **Job: validate** (runs on `ubuntu-22.04`):
       - Checkout, setup Node 20 (with npm cache), install Rust stable
       - Install Linux system deps: `libwebkit2gtk-4.1-dev libappindicator3-dev librsvg2-dev patchelf` (needed for Cargo check on Linux)
       - `npm ci`
       - `npx tsc --noEmit` (TypeScript type check)
       - `npm run test` (Vitest unit tests)
       - `cd src-tauri && cargo check` (Rust compilation check)
       - `cd src-tauri && cargo clippy -- -D warnings` (Rust linting)
       - Use `swatinem/rust-cache@v2` for Rust caching
    3. **Job: e2e** (runs on `ubuntu-22.04`, only on PRs):
       - Same setup as validate + full Tauri build
       - `xvfb-run npm run test:e2e` (E2E tests with virtual display)
       - Depends on `validate` job (only runs if validate passes)
       - Note: E2E tests require a built binary, so this job is heavier

    CI pipeline optimizations (Claude's discretion):
    - Use npm cache via `actions/setup-node` cache option
    - Use `swatinem/rust-cache@v2` with `workspaces: src-tauri -> target`
    - `fail-fast: false` if using matrix
    - E2E only on PRs (not every push) to save CI minutes
  </action>
  <verify>
    `cat .github/workflows/ci.yml` shows valid YAML with validate and e2e jobs.
    `grep "tsc --noEmit" .github/workflows/ci.yml` exists.
    `grep "cargo clippy" .github/workflows/ci.yml` exists.
    `grep "xvfb-run" .github/workflows/ci.yml` exists.
  </verify>
  <done>CI workflow validates TypeScript, unit tests, Cargo check, and Clippy on every push/PR. E2E tests run on PRs with xvfb.</done>
</task>
