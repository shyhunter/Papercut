    1. Add Rust dependencies in `src-tauri/Cargo.toml`:
       ```toml
       tauri-plugin-updater = "2"
       tauri-plugin-process = "2"
       ```
    2. Register plugins in `src-tauri/src/lib.rs` in the existing plugin chain:
       ```rust
       .plugin(tauri_plugin_updater::Builder::new().build())
       .plugin(tauri_plugin_process::init())
       ```
    3. Add capabilities in `src-tauri/capabilities/default.json`:
       - `"updater:default"`
       - `"process:allow-restart"`
    4. Add updater configuration in `src-tauri/tauri.conf.json` under `plugins`:
       ```json
       "updater": {
         "endpoints": [
           "https://github.com/USER/papercut/releases/latest/download/latest.json"
         ],
         "pubkey": ""
       }
       ```
       Note: The pubkey will be populated later when `tauri signer generate` is run. Leave empty for now — the updater will gracefully fail without a valid key, which is fine during development.
       Note: Replace USER with the actual GitHub username/org — or leave as placeholder with a TODO comment.
    5. Install JS packages:
       ```bash
       npm install @tauri-apps/plugin-updater @tauri-apps/plugin-process
       ```
  </action>

  <action>
    1. Create `src/components/UpdateChecker.tsx`:

         - "Update available: v{version}" with "Update Now" and "Later" buttons
       - On "Update Now":
         - Show download progress (use the progress callback from `downloadAndInstall`)
         - After install, call `relaunch()` from `@tauri-apps/plugin-process`

       - Style with Tailwind, support dark mode via existing theme vars
       - Do NOT block the app — update check runs in background

    2. Integrate in `src/App.tsx`:
       - Render `<UpdateChecker />` after the splash screen completes (not during splash)
       - Place it at the top of the app layout (before Dashboard/tool content)
       - It should render nothing when no update is available (return null)
