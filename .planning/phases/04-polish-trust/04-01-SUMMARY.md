## Decisions Made

- LazyStore always requires explicit `.save()` call after `.set()` — no auto-persist. Missing this loses data silently between sessions.
- `fs:allow-exists` must include an `allow` scope (`$HOME/**`) or Tauri throws a forbidden path error at runtime — no fallback.
- Store plugin registered before fs plugin in the `.plugin()` chain — Tauri processes plugins in registration order.
- Inline error (not toast) for invalid drops matches the plan spec; inline feedback is contextually appropriate for the drop zone.
- Corrupt-file useEffect watchers intentionally omit exhaustive deps (eslint-disable-next-line) to match the existing useEffect pattern in App.tsx that does the same for step-advance effects.
